# BUG-011: Lembretes criados sem data armazenam formato incorreto e causam ordenação errada

## Descrição do problema

Lembretes criados sem informar explicitamente a `data_lembrete` têm a data armazenada no formato `DD/MM/AAAA HH:MM:SS` em vez do padrão ISO `AAAA-MM-DD HH:MM:SS`. Isso causa **inconsistência de formatos** no arquivo JSON e **ordenação temporal incorreta** na listagem.

- Criar um lembrete passando `data_lembrete` explicitamente funciona normalmente (formato ISO)
- Criar um lembrete sem informar `data_lembrete` armazena a data no formato brasileiro
- A data aparece visualmente correta na tela (ex: "05/01/2026 14:30:00"), mas o formato é diferente dos demais
- A listagem de lembretes fica fora de ordem quando há registros com os dois formatos misturados
- Qualquer processamento que use `strtotime` sobre essas datas retorna resultado errado (interpreta dia como mês)

## Resultado esperado

- Todos os lembretes devem ter `data_lembrete` armazenada no formato `AAAA-MM-DD HH:MM:SS`
- O formato deve ser consistente independentemente de a data ter sido informada pelo usuário ou gerada automaticamente
- A ordenação por data na listagem deve funcionar corretamente para todos os registros
