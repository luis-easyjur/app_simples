# BUG-008: Agendas com horários consecutivos são rejeitadas como conflito

## Descrição do problema

No módulo de Agendas, ao tentar cadastrar uma agenda cujo horário de início coincide exatamente com o horário de fim de outra agenda já existente, o sistema rejeita o cadastro com a mensagem **"Já existe agenda neste horário"**.

- Criar uma agenda das 09:00 às 10:00 funciona normalmente
- Ao tentar criar uma segunda agenda das 10:00 às 11:00 (no mesmo dia), o sistema retorna erro
- O erro só ocorre quando os horários se tocam exatamente nas bordas
- Com qualquer intervalo entre as agendas (ex: 10:01 às 11:00), o cadastro funciona normalmente

## Resultado esperado

- Agendas com horários consecutivos (sem sobreposição real) devem ser permitidas
- Apenas agendas com intervalos que se sobrepõem de fato devem ser bloqueadas
- Exemplo válido: 09:00–10:00 e 10:00–11:00 devem coexistir sem conflito
