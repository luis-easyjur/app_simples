# BUG-009: Nova pessoa recebe ID já utilizado após exclusão, corrompendo agendas

## Descrição do problema

Após excluir uma pessoa e cadastrar uma nova, a nova pessoa recebe um ID que já foi utilizado anteriormente. Isso causa uma **colisão silenciosa de referências**: agendas que pertenciam à pessoa excluída passam a exibir o nome da nova pessoa.

- Cadastrar 3 pessoas (IDs 1, 2, 3) funciona normalmente
- Excluir a pessoa de ID 2 funciona normalmente
- Ao cadastrar uma nova pessoa, ela recebe o ID 2 (que já foi usado)
- Agendas associadas à antiga pessoa ID 2 agora exibem o nome da nova pessoa
- Nenhuma mensagem de erro é exibida em nenhuma etapa

## Resultado esperado

- O ID gerado para uma nova pessoa deve sempre ser maior que o maior ID já existente
- A exclusão de registros não deve interferir na sequência de IDs futuros
- Agendas existentes devem continuar exibindo os dados corretos após operações de exclusão e criação
