# BUG-001: Validação incorreta de data no cadastro de agendas

## Descrição do problema

Ao cadastrar uma nova agenda no sistema, a validação de data está se comportando de forma invertida:

- **Datas no passado** são aceitas normalmente
- **Datas de hoje ou futuras** retornam erro: "Data não pode ser no passado"

Isso impede o cadastro de agendas para reuniões, audiências ou compromissos futuros, que é o uso esperado do sistema.

## Resultado esperado

- Datas **no passado** devem ser **rejeitadas** com mensagem de erro
- Datas de **hoje** e **futuras** devem ser **aceitas** normalmente
- O usuário deve conseguir agendar compromissos para datas futuras sem erro
